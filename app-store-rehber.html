Yayında!
GitHub Actions neden kullanıyoruz? Xcode sadece Mac'te çalışır. Uygulamayı build etmek (derleme) için Mac gerekli. Ama kendi Mac'ine sahip olmak zorunda değilsin — GitHub, build işlemini kendi Mac sunucusunda ücretsiz olarak koşturur. Sen sadece kodu push edersin, gerisini GitHub halleder. Başlamadan Önce — Ön Koşullar
🌐 Herhangi bir tarayıcı

Aşağıdakilerin hepsine sahip olduğundan emin ol. Eksik olan varsa önce onu tamamla.

✅ Apple Developer Hesabı

developer.apple.com — yıllık $99 ücretli plan aktif olmalı. Yoksa: developer.apple.com → Enroll.

✅ App Store Connect Erişimi

appstoreconnect.apple.com — Apple ID ile giriş yapılabiliyor olmalı. Genellikle otomatik aktif.

✅ GitHub Repo

Proje GitHub'da mevcut ve .github/workflows/ios-deploy.yml dosyası var.

✅ Mac'e Kısa Erişim

Kendi Mac'in veya bir arkadaşının Mac'i — yaklaşık 10 dakika yeterli. Sadece sertifika adımı için.

ℹ️
Mac'e hiç erişimin yoksa: Xcode Cloud kullanabilirsin. App Store Connect → Xcode Cloud → ücretsiz 25 saat/ay ile hem build hem signing otomatik yapılır. Ama bu rehber GitHub Actions yolunu anlatıyor çünkü daha esnek.
01
Bundle ID Oluştur (App ID)
🌐 Tarayıcı developer.apple.com/account

Bundle ID, uygulamanın benzersiz kimliğidir. com.uzmandiyet.app şeklinde tanımlanmış. Bunu Apple sistemine kaydetmemiz gerekiyor.

1
Developer Portal'a Giriş
developer.apple.com/account adresine git. Sağ üst köşede Apple ID'nle giriş yap.
2
Certificates, IDs & Profiles'a Git
Sol menüden Certificates, IDs & Profiles'a tıkla.
3
Identifiers → App IDs
Sol menüden Identifiers'a tıkla → Sağ üstteki mavi + butonuna bas → App IDs seç → Continue → App seç → Continue.
4
Bilgileri Doldur
Description: UzmanDiyet
Bundle ID: Explicit seç → com.uzmandiyet.app yaz
Capabilities: Push Notifications'ı işaretle (ileride lazım olur)
→ Continue → Register
✅
Bu adım tamamen tarayıcıdan yapılır. Mac gerekmez.
02
Distribution Sertifikası Oluştur ve P12 Export Et
🍎 Mac Gerekli ~10 dakika

Bu adım için Mac'e ihtiyaç var çünkü CSR (Certificate Signing Request) dosyası ve P12 export işlemi Mac'in Keychain uygulamasıyla yapılır. Bir arkadaşının Mac'ini bile kullansan yeterli — işlem 10 dakika sürer.

A) CSR Dosyası Oluştur (Mac)

1
Keychain Access'i Aç
Mac'te Spotlight'a (⌘ + Space) Keychain Access yaz, aç.
2
Certificate Assistant
Üst menü → Keychain Access → Certificate Assistant → Request a Certificate from a Certificate Authority…
3
Bilgileri Doldur
User Email Address: Apple ID e-posta adresin
Common Name: UzmanDiyet
Request is: Saved to disk seç
→ Continue → Masaüstüne kaydet (CertificateSigningRequest.certSigningRequest)

B) Developer Portal'da Sertifika Oluştur (Tarayıcı)

4
Certificates → + → Apple Distribution
developer.apple.com/account → Certificates, IDs & Profiles → Certificates → mavi + → Apple Distribution seç → Continue.
5
CSR Dosyasını Yükle
Choose File'a tıkla → Az önce masaüstüne kaydettiğin CertificateSigningRequest.certSigningRequest dosyasını seç → Continue → Download → distribution.cer masaüstüne indi.
6
Sertifikayı Keychain'e Ekle
Mac'te indirilen distribution.cer dosyasına çift tıkla. Keychain Access otomatik açılır ve sertifikayı ekler.
7
P12 Olarak Export Et
Keychain Access'te My Certificates sekmesine tıkla → Apple Distribution: … sertifikasını bul → sağ tıkla → Export "Apple Distribution…" → Format: Personal Information Exchange (.p12) → Kaydet → Şifre belirle (örn: Build2024!) — bu şifreyi not al!
⚠️
P12 şifresini mutlaka not al. Bu şifreyi GitHub Secrets'a girmen gerekecek (P12_PASSWORD). Unutursan sertifikayı yeniden oluşturman gerekir.
💡
P12 dosyasını Windows'a aktar: Mac'ten USB ya da AirDrop ile değil, e-posta ile kendine gönder ya da iCloud Drive'a at. Windows'ta indir, masaüstüne kaydet.
03
App Store Provisioning Profile Oluştur
🌐 Tarayıcı developer.apple.com/account

Provisioning Profile, sertifikayı ve Bundle ID'yi birbirine bağlayan bir dosyadır. Bu adım tamamen tarayıcıdan yapılır.

1
Profiles → +
developer.apple.com/account → Certificates, IDs & Profiles → Sol menüden Profiles → mavi + butonu.
2
App Store Connect Seç
Distribution başlığı altında App Store Connect seç → Continue.
3
Bundle ID Seç
Dropdown'dan com.uzmandiyet.app seç → Continue.
4
Sertifikayı Seç
Az önce oluşturduğun Apple Distribution sertifikasını işaretle → Continue.
5
İsim Ver ve İndir
Provisioning Profile Name: UzmanDiyet AppStore yaz → Generate → Download. Dosya adı: UzmanDiyet_AppStore.mobileprovision
✅
Bu dosyayı da Windows'a aktar (e-posta / iCloud / USB). P12 ile birlikte masaüstüne kaydet.
04
App Store Connect API Key Oluştur
🌐 Tarayıcı appstoreconnect.apple.com

GitHub Actions, App Store Connect'e IPA yüklemek için bu API Key'i kullanır. Tamamen tarayıcıdan yapılır.

1
App Store Connect → Users & Access
appstoreconnect.apple.com → Sol üst menüden Users and Access tıkla.
2
Integrations → App Store Connect API
Üst sekmelerden Integrations → App Store Connect API'ye tıkla.
3
Generate API Key
Mavi Generate API Key butonu (veya + ikonu) → Name: GitHub Actions → Access: App Manager seç → Generate.
4
Bilgileri Not Al ve Dosyayı İndir
Key oluşturuldu. Şu değerleri not al:

• Key ID (örn: ABC123XYZ) — sayfada görünür
• Issuer ID — sayfanın üstünde "Issuer ID" başlığı altında

Download API Key'e tıkla → AuthKey_ABC123XYZ.p8 inecek.
⚠️ Bu dosya sadece bir kez indirilebilir. Kaybetme!
⚠️
.p8 dosyası kritiktir. "Download API Key" butonuna bir kez tıklanabilir. İndirdikten sonra Windows'ta güvenli bir yere kaydet. Kaybedersen yeni key oluşturmak zorundasın.
05
Dosyaları Base64'e Çevir
🪟 Windows PowerShell

GitHub Secrets sadece metin (string) kabul eder. P12, .mobileprovision ve .p8 dosyaları binary dosyalardır, doğrudan yapıştırılamaz. Bu yüzden Base64 formatına çeviriyoruz. Base64, binary dosyaları metin olarak temsil etmenin standart yoludur.

Windows PowerShell'i aç (Win + X → Windows PowerShell) ve şu komutları çalıştır:

PowerShell Her komut sonucu panoya kopyalar
# 1. P12 sertifikasını base64'e çevir ve panoya kopyala [Convert]::ToBase64String([IO.File]::ReadAllBytes("C:\Users\Win10\Desktop\distribution.p12")) | clip # → Şimdi Ctrl+V ile GitHub'a yapıştırabilirsin (BUILD_CERTIFICATE_BASE64) # 2. Provisioning Profile'ı base64'e çevir [Convert]::ToBase64String([IO.File]::ReadAllBytes("C:\Users\Win10\Desktop\UzmanDiyet_AppStore.mobileprovision")) | clip # → Ctrl+V ile yapıştır (BUILD_PROVISION_PROFILE_BASE64) # 3. .p8 API Key'i base64'e çevir [Convert]::ToBase64String([IO.File]::ReadAllBytes("C:\Users\Win10\Desktop\AuthKey_ABC123XYZ.p8")) | clip # → Ctrl+V ile yapıştır (APP_STORE_CONNECT_KEY_BASE64)
💡
Dosya yolu farklıysa komutu ona göre düzenle. Dosya adları birebir aynı olmak zorunda değil, sadece gerçek dosyanın yolunu yaz.
06
GitHub Secrets Kur
🌐 Tarayıcı github.com → Repo → Settings → Secrets

GitHub Actions, build sırasında bu "secret" değerlere ihtiyaç duyar. Bunlar şifreli saklanır, ne sen ne başkası düz metin olarak göremez.

1
Repo Ayarlarına Git
github.com → Repon'a git → üst menüden Settings → sol menüden Secrets and variables → Actions.
2
New Repository Secret
Yeşil New repository secret butonuna tıkla → Name alanına secret adını yaz → Secret alanına değeri yapıştır → Add secret. Her secret için tekrar et (9 adet).
# Secret Adı Değer / Nereden
1 BUILD_CERTIFICATE_BASE64 P12 dosyasının base64 çıktısı (Adım 5, komut 1)
2 P12_PASSWORD P12 export sırasında belirlediğin şifre (örn: Build2024!)
3 KEYCHAIN_PASSWORD Herhangi bir şifre uydur — GitHub Actions geçici keychain için kullanır. Örn: CI-keychain-2024
4 BUILD_PROVISION_PROFILE_BASE64 .mobileprovision dosyasının base64 çıktısı (Adım 5, komut 2)
5 PROVISIONING_PROFILE_NAME Profile'ın tam adı: UzmanDiyet AppStore
6 APPLE_TEAM_ID developer.apple.com/account → sağ üst köşe → adının altında Team ID (10 karakter, örn: A1B2C3D4E5)
7 APP_STORE_CONNECT_KEY_ID Adım 4'te not aldığın Key ID (örn: ABC123XYZ)
8 APP_STORE_CONNECT_ISSUER_ID Adım 4'te not aldığın Issuer ID (UUID formatında, örn: a1b2c3d4-…)
9 APP_STORE_CONNECT_KEY_BASE64 .p8 dosyasının base64 çıktısı (Adım 5, komut 3)
✅
9 secret de eklendikten sonra sayfada 9 satır göreceksin. Değerleri göremezsin ama adları listede görünür — hepsinin orada olduğunu doğrula.
07
Build Tetikle — GitHub Actions Başlat
🪟 Windows PowerShell

Workflow dosyası v* formatında bir tag push edilince otomatik tetikleniyor (örn: v1.0.0). Bu komutu Windows'ta proje klasöründe çalıştır.

PowerShell C:\Users\Win10\Desktop\Masterclass klasöründe çalıştır
# Değişiklikler commit edilmiş mi? Kontrol et git status # Varsa commit et git add . git commit -m "iOS: v1.0.0 hazırlık" git push origin main # Tag oluştur ve push et — bu komut GitHub Actions'ı tetikler git tag v1.0.0 git push origin v1.0.0
→
Actions Sekmesini Takip Et
github.com → Repon → Actions sekmesi → "iOS — App Store Deploy" workflow'u görürsün → Tıkla → Adım adım ilerlemeyi canlı izleyebilirsin.
Toplam süre: ~30–45 dakika (CocoaPods kurulumu uzun sürer).
→
Build Başarılı Olursa
Son adım "Upload to App Store Connect" yeşil ✓ olursa IPA yüklendi demektir. App Store Connect'te uygulamanın sayfasına git → TestFlight sekmesinde birkaç dakika içinde Processing durumunda görürsün.
→
Build Başarısız Olursa
Kırmızı ✗ olan adıma tıkla → log'a bak → hata mesajını oku. En sık hatalar: yanlış secret adı, yanlış Profile ismi, süresi dolmuş sertifika.

App Store'a Gönder (Submit for Review)
🌐 Tarayıcı appstoreconnect.apple.com

IPA yüklendikten sonra App Store Connect'te bilgileri doldurman ve Apple'a incelemeye gönde rmen gerekiyor.

1
Uygulamanın Sayfasına Git
appstoreconnect.apple.com → My Apps → UzmanDiyet → iOS versiyonu listesini gör.
2
Ekran Görüntüleri Yükle (Zorunlu)
App Information → Previews and Screenshots bölümüne ekran görüntüleri yükle.

Zorunlu boyutlar:
• iPhone 6.7" (iPhone 14 Pro Max): 1290 × 2796 px
• iPhone 6.5" (iPhone 11 Pro Max): 1242 × 2688 px
• iPad Pro 12.9": 2048 × 2732 px (isteğe bağlı ama önerilir)

Ekran görüntüsü almak için: iPhone Simulator (Mac'te Xcode ile) veya gerçek cihaz. Yoksa: placehold.co ile sahte görsel de kabul edilir ama Apple reddedebilir.
3
Uygulama Bilgilerini Doldur
App Information sayfasında:
• Name: UzmanDiyet
• Subtitle: AI Destekli Diyet Asistanı
• Privacy Policy URL: Zorunlu (uzmandiyet.com/gizlilik gibi bir sayfa gerekli)
• Category: Health & Fitness → Medical (veya Food & Drink)
• Age Rating: Soruları yanıtla (genellikle 4+ çıkar)
4
Versiyon Açıklaması Yaz
Description: Uygulamanın ne işe yaradığını Türkçe anlat (max 4000 karakter)
Keywords: diyet, kalori, yapay zeka, diyetisyen, beslenme (max 100 karakter)
Support URL: uzmandiyet.com veya bir iletişim sayfası
What's New: İlk sürüm için: "UzmanDiyet'in ilk sürümü."
5
Build Seç
Sayfanın Build bölümünde + ikonuna tıkla → GitHub Actions'ın yüklediği build'i seç (Processing bitmişse görünür). Processing ortalama 10–15 dakika sürer.
6
Submit for Review
Sağ üstteki Add for Review → ardından Submit to App Review butonuna tıkla. Birkaç soru sorulur (ihracat uyumluluğu, içerik hakkı vb.) — yanıtla.

Apple incelemesi: 1–3 iş günü. E-posta ile bildirim gelir.
⚠️
Privacy Policy zorunlu. Uygulama sağlık verileri işlediği için Apple kesinlikle Privacy Policy URL istiyor. Yayına çıkmadan önce uzmandiyet.com/privacy gibi basit bir sayfa oluştur.
✓
